# docs/modules Directory

## Purpose
Module catalog documentation linking blueprint metadata to deployment state.

## How to Use
Describe module purpose, fee tier, NFT requirements, tests, and UI references.

## Naming Conventions
module-social-share.md, module-defi-lend.md.

## Coding Rules
Include YAML front matter so automation can parse status.

## Scaling Notes
One file per module ensures clarity even with 1000 entries.

## Security Notes
Note any permissions or trusted third parties used by the module.

## Codex/Cursor Automation
Registry sync job updates doc status from MODULE_REGISTRY.json.
